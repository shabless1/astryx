# Part 5 · The Engine Layers

The Astryx engine has six layers, each adding diagnostic intelligence:

## Layer 1 — Natal Chart Calculation

NASA-grade ephemeris (astronomy-engine library, Don Cross / Microsoft) calculates sub-arcsecond precision for all 10 planets + Ascendant + Midheaven. Pure TypeScript, no native binaries, works in browser and on Vercel.

**Tropical zodiac. Placidus houses. 8 aspect types detected with appropriate orbs.**

If birth time is unknown, the engine switches to **Solar Chart mode** (Sun on Ascendant) — a standard professional technique. Labeled clearly with `☉ Solar Chart` badge throughout.

## Layer 2 — Transit Engine (the daily-return mechanism)

Calculates today's sky positions for all 10 planets. Detects aspects to natal chart with tight transit orbs (2° for major aspects, 1.5° for minor). Returns top 7 ranked by weight.

**Weighting algorithm:** Outer planets matter more in transit (their effects compound over years). Pluto/Neptune/Uranus/Saturn dominate. Aspect class × exactness × applying/separating × planetary weight = ranking score.

**Days-to-exact** calculated for every active transit so the user sees: *"4 days to exact, applying — intensifying"* or *"6 days past, separating — fading."*

This is the daily-return mechanism. Every time the user opens Astryx, fresh transit data populates the diagnostic. The headline shifts with the sky.

## Layer 3 — Symptom Routing

User reports symptom (e.g., "anxiety"). Engine normalizes to root-cause key via keyword map. Looks up in `medicalAstrology.rootCauseIndex`. Parses candidate planetary signatures from natural-language strings. Scores each signature against the user's natal chart + active transits.

**Scoring weights:**
- +30 if planet is in dominant pattern
- +20 per active transit where planet is transiting another natal point
- +15 per active transit where planet is being hit
- +25 if signature's two planets are in a mutual natal aspect

Highest-scoring signature wins. Returns full diagnostic with evidence trail.

**10 root-cause categories** at launch: anxiety, depression, money block, exhaustion/burnout, inflammation, hormonal imbalance, digestive, sleep, skin, fertility. Each has 3-5 subtype patterns mapped to specific planetary signatures.

## Layer 4 — Diagnostic Block (the car-diagnostic translation)

Cross-references medicalAstrology.json (planet → anatomy → root cause) with cellSalts.json (sign → mineral prescription + Bonacci gestation rule) with current transits with reported symptoms.

Returns the **three-layer translation**:
1. **Cosmic Layer** — what's happening astrologically
2. **Body Layer** — what it does in your anatomy (pulled from medicalAstrology symptomSignatures.whenStressed.headline)
3. **Action Layer** — what to do (pulled from medicalAstrology plainLanguageBridge.howToRestore)

Plus the cell salt prescription, anatomy block, symptom signatures, and plain-language bridge.

## Layer 5 — Unified Prescriptions

For each unique planetary signature (dominant + symptom-routed + transit-driven, deduplicated by planet), generates ONE cohesive prescription card that bundles all 5 senses + crystal + cell salt + botanical + tuning fork — all keyed to the same planet.

Plus an **integration paragraph** that walks the user through weaving them together in a 20-30 minute session.

Plus all relevant safety notes (with mandatory Malachite warning when applicable).

## Layer 6 — Compliance Gate

Every output-facing string passes through the compliance enforcement layer:
- Banned phrase guard (7 banned phrases — diagnose, treat, cure, prescribe, guarantee, etc.)
- Approved phrasing wrapper (per tier: individual gets plain language + referral framing; practitioner gets clinical terminology + classical citations)
- Crisis keyword detection (self-harm, medical emergency, mental health crisis, domestic violence) — pauses normal flow, displays Crisis Resources card
- Disclaimer attachment (universal block + persistent micro)

# Part 9 · Compliance & Scope of Practice

## The Three Hard Rules

1. **Astryx is the instrument, not the diagnostician.** We point to indicators. The user — or their licensed practitioner — decides what to do.

2. **All output language is probabilistic.** "May suggest." "May indicate." "May correlate with." "Classical sources associate this with." Never "you have." Never "this causes." Never "this cures."

3. **The disclaimer never disappears.** Persistent micro-disclaimer on every screen footer (`ⓘ Reference tool · Not medical advice`). Full disclaimer block on every output screen + every PDF export.

## The 7 Banned Phrasings

Hard-coded in `lib/compliance.ts` — flagged by linter on any output string:
- "you have" / "you are suffering from"
- "this causes" / "the cause is"
- "diagnose" / "diagnosis" (when applied to user)
- "treat" / "treatment" / "cure"
- "prescribe" / "prescription" (medical sense)
- "guarantee" / "guaranteed"
- "safe for everyone" / "no side effects"

## The 12 Approved Phrasings

The replacements:
- "may suggest" · "may indicate" · "may correlate with"
- "is classically associated with"
- "the chart indicates"
- "this combination has historically been linked to"
- "classical sources associate this with"
- "consider exploring with your practitioner"
- "for reference"
- "observational pattern"
- "may benefit from"
- "general wellness alignment"

## Crisis Keyword Handling

Every text intake passes through `detectCrisis()` before processing:
- **Self-harm** keywords (suicide, kill myself, end it all, self-harm) → pause flow, display Crisis Resources card with 988 Lifeline, Crisis Text Line
- **Medical emergency** (can't breathe, chest pain, stroke symptoms, severe bleeding, overdose) → same pause, direct to 911
- **Acute mental health** (psychosis, hallucinating, breaking down) → same pause, mental health emergency framing
- **Domestic violence** (he's hurting me, abuse, scared of partner) → same pause, DV hotline

The Crisis Resources card is **non-negotiable** and overrides normal diagnostic flow.

## The Attestation System

Per COMPLIANCE.md §7:

- **Individual** signs at signup: 18+, understands observational reference only, will consult healthcare provider for persistent symptoms, will not stop prescribed medication based on Astryx output
- **Practitioner** (self-attested): selects modality, attests under penalty of perjury to licensure/certification in that modality, accepts professional responsibility, will obtain client informed consent
- **Verified Practitioner**: uploads license/certification, additional attestation that documents are authentic and current

All attestations are **versioned, timestamped, IP-stamped, and stored permanently** as legal armor.

## Critical Referral Rules

Every body system file has a `criticalReferralRules` array — explicit referral triggers per system. Examples:
- **Cardiovascular:** Chest pain at rest → 911. BP > 180/120 with symptoms → emergency.
- **Respiratory:** Acute severe dyspnea → 911. Hemoptysis → urgent MD.
- **Nervous:** Sudden focal weakness → 911 (FAST stroke protocol). Suicidal ideation → 988.
- **Reproductive:** Postmenopausal bleeding → urgent gyn (always abnormal).
- **All systems:** Have explicit red flag keyword lists.
